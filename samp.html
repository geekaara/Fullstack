VERSIONS OF HTML

HTTP has four versions — HTTP/0.9, HTTP/1.0, HTTP/1.1, and HTTP/2.0. Today the version in common use is HTTP/1.1 and the future will be HTTP/2.0.

HTTP/0.9

It supported only GET method, the response type is only hypertext, the connection terminated immediately after the response and No HTTP, No status/error codes, No URLs, No versioning.

HTTP/1.0

Methods supported are GET , HEAD , POST, the connection terminated immediately after the response, responses include hypertext, scripts, stylesheets and media and it has fields for metadata

HTTP/1.1

Introduced critical performance optimizations and feature enhancements like persistent and pipelined connections, chunked transfers, compression/decompression, content negotiations, virtual hosting, faster response and great bandwidth savings by adding cache support.

Methods supported are GET , HEAD , POST , PUT , DELETE , TRACE , OPTIONS Connection nature is long-lived

WHAT HAPPENS WHEN YOU TYPE A URL IN THE ADDRESS BAR IN THE BROWSER

 

You enter a URL into a web browser. The browser looks up the IP address for the domain name via DNS. The browser sends a HTTP request to the server. The server sends back a HTTP response. The browser begins rendering the HTML. The browser sends requests for additional objects embedded in HTML (images, css, JavaScript). Once the page is loaded, the browser sends further async requests as needed.
